Castellyn — мульти-мониторный вывод сессий: мокапы

Твои 3 монитора: большой (16:9), средний (16:9), маленький длинный (21:9). Терминалы — те же PTY/xterm, что уже есть в Castellyn.

Во всех вариантах SSH-сессия — это просто новый тип сессии: системный ssh user@host в той же панели-терминале (DRY, как в нашей сетке). Отличие вариантов — как именно сессии раскидываются по мониторам.

Вариант A — Вынос панели на монитор

Рекомендую

У каждой панели в шапке кнопка ⬈ → «на Монитор 2/3». Клик — и эта сессия открывается безрамочным дочерним окном на выбранном мониторе. Тот же терминал, просто переехал в своё окно. Можно выносить по одной, в любой момент вернуть назад.

Castellyn · Sessions
shell · E:\Scripts⬈
PS> git status
✓ clean
PS> _
claude · ccmy⬈
● ready
> build the grid
…
ssh · srv1⬈ →М2
dansc@srv1:~$ htop
load 0.4
ssh · srv2⬈ →М3
dansc@srv2:~$ logs
warn x2
Монитор 1 — главное окно (все панели)
ssh · dansc@srv1
dansc@srv1:~$ docker ps
NAMES STATUS
sourcevault Up 3d
slidio-web Up 3d
dansc@srv1:~$ _
Монитор 2 — вынесенная сессия srv1
ssh · dansc@srv2
dansc@srv2:~$ tail -f app.log
12:01 request ok
12:02 warn: retry
12:03 request ok _
Монитор 3 — вынесенная сессия srv2
Плюсы
  • Максимальный DRY: тот же xterm/PTY, события pty:data:<id> глобальные — окну ничего не дублируем
  • Инкрементально: по одной панели, в любой момент назад
  • Ближе всего к «как в сетке», но внутри Castellyn
Минусы
  • Дочернее окно = ещё один webview (немного памяти на окно)
  • Надо запоминать раскладку вынесенных окон между запусками
UX: кнопка ⬈ на каждой панели → «на Монитор N»

Вариант B — Рабочий стол на каждый монитор

Полный воркспейс

Кнопка «Разнести по мониторам» открывает на каждом мониторе по полноценному окну-сетке (как сама вкладка Sessions), и сессии распределяются по ним. Получается мульти-мониторное рабочее пространство целиком.

Castellyn · Sessions
shell
PS> _
claude · ccmy
● ready
Монитор 1 — Sessions (главное)
Castellyn · Sessions #2
ssh·srv1
srv1$ htop
ssh·srv2
srv2$ logs
ssh·srv3
srv3$ top
shell
PS>
Монитор 2 — окно-сетка #2
Sessions #3
ssh
a$
ssh
b$
ssh
c$
Монитор 3 — окно-сетка #3
Плюсы
  • Настоящий мульти-мониторный воркспейс: на каждом мониторе полноценная сетка
  • Хорошо при многих сессиях разом
Минусы
  • Больше кода: управление N окнами, их колонками и состоянием
  • Сложнее раскладка/восстановление; тяжелее, чем нужно для «вынести пару сессий»
UX: одна кнопка «Разнести по мониторам»

Вариант C — Тайлить как внешнюю сетку

Как grid-main

Кнопка «Тайлить сессии» открывает сессии отдельными ОС-окнами, расставленными по мониторам ровно как наш grid-main.ps1. Максимально «как мы сделали», но это уже внешние терминалы, а не панели Castellyn — теряется встроенный xterm/UI приложения.

ssh·a
a$
ssh·b
b$
shell
PS>
Монитор 1 — отдельные ОС-окна
ssh·c
c$
ssh·d
d$
Монитор 2 — отдельные ОС-окна
ssh·e
e$
ssh·f
f$
Монитор 3 — отдельные ОС-окна
Плюсы
  • 1:1 как наша готовая сетка — переиспользуем логику расстановки по мониторам/DPI
  • Терминалов может быть много, тайлятся вплотную
Минусы
  • Это внешние окна, не панели Castellyn — теряется встроенный xterm, заголовки, поиск, тема приложения
  • Дублирование с grid-main; зачем тогда внутри Castellyn
UX: одна кнопка «Тайлить сессии»
