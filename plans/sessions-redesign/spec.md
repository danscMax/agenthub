# Дизайн-спек: редизайн панели «Параллельные сессии»

Статус: **утверждён** (2026-06-23). Макеты: `mockups.html` (20 вариантов), `design.html` (финал).

## Модель

Сессия = **среда** × **место** × **папка** × **аргументы**.

- **Среда** (`tool`): `claude` | `opencode` | `shell`. (Значение `ssh` как «инструмент» **упраздняется** — см. миграцию.)
- **Место** (`sshTarget`): `null` = этот ПК (локально); строка `user@host …` = по SSH.
- **Папка** (`cwd` локально / `remoteDir` по SSH).
- **Аргументы** (`args`): только для claude/opencode.
- **Профиль** (`profile`): только для claude.

Среда и место **ортогональны**: любую среду можно запустить локально или по SSH.
- `shell` + SSH = вход на сервер (прежнее «SSH»).
- `claude`/`opencode` + SSH = запуск этой команды **на сервере** (должна быть установлена/авторизована там). Реализация для Windows/PowerShell-сервера — через тот же `ssh -t <target> powershell -NoExit -EncodedCommand <…>`, что уже есть для удалённой папки.

## UI (design.html)

1. **Сегмент сред** сверху: `[▶ Claude] [▶ opencode] [▶ shell]` со SVG-иконками.
2. **Фраза-конструктор** под сегментом, читается как предложение и подстраивается:
   `Запустить {среда} [профиль {p}] на {место ▾} в {папка ▾} [с {args ▾}] [★] [▶ Запустить]`.
3. **Переключатель «на …»**: дропдаун `этот ПК / 🖥 сервер` + авто-статус (🟢/🔴, `test_ssh_host`) + «＋ Добавить сервер».
4. **Папка ▾**: Избранное / Недавние / **Проекты** (подпапки «корня проектов») / Обзор. По SSH — сохранённые удалённые пути (листать нельзя).
5. **Избранное (ярлыки)**: ★ закрепляет всю фразу → чип в 1 клик. Растёт из конструктора и из запущенной панели.
6. **Наборы**: несколько сессий разом; «Запустить все профили» становится одним из наборов.
7. **Настройки (⚙)**: корень проектов, папка/аргументы по умолчанию, колонки, автопроверка. Тумблер **«Спрашивать папку» убран** (папка всегда видна во фразе).

## Бэкенд

`session_spawn` получает `ssh_target: Option<String>` (камелкейс `sshTarget`).
- `ssh_target=None` → текущее локальное поведение.
- `ssh_target=Some` + среда:
  - собрать внутреннюю PS-строку = склейка `["Set-Location -LiteralPath '<dir>'" если dir, "<tool> <args>" если tool≠shell]` через `; `.
  - пусто (shell без папки) → `ssh -t <target>`; иначе → `ssh -t <target> powershell -NoExit -EncodedCommand <b64(inner)>`.
- Переиспользуем `ps_encoded_command`.

## Фазы (каждая — зелёные гейты: cargo / check / i18n / vitest)

1. **Бэкенд (аддитивно):** `session_spawn` + `sshTarget`; поддержка «среда по SSH»; `tool='ssh'` пока живёт (легаси). `ipc.ts` сигнатура. Не ломает фронт.
2. **Компоненты лаунчера:** `EnvSegment` (SVG-иконки), `LaunchPhrase` (фраза), `LocationPicker` (место+статус), переиспользовать `Select`/`DropdownMenu`/`FolderField`.
3. **Папка/проекты:** дропдаун папки с Проектами из корня; удалённые пути для SSH.
4. **Избранное + Наборы:** модель ярлыков (env+location+folder+args), закрепление ★, чипы; «Запустить все» → набор.
5. **Настройки (⚙):** панель; убрать тумблер «Спрашивать папку».
6. **Миграция/чистка:** перевести quickSsh/диалог/workspaces на новую модель; убрать `tool='ssh'`; обновить заголовок панели (`среда · место`).
7. **Полировка:** i18n ru/en/zh, SVG-иконки, сборка, прогон гейтов.
