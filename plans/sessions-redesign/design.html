Castellyn · «Параллельные сессии» — финальный дизайн

Базис: инлайн-конструктор фразы (№20) + сегмент сред сверху (№8). Ключевая идея: SSH — это переключатель «где», а не среда. Модель запуска: среда (Claude / opencode / shell) × место (этот ПК / SSH-сервер) × папка × аргументы.

1 Главный лаунчер

Сегмент сред — сверху (быстрое переключение). Под ним фраза, которая читается как предложение и подстраивается под выбранную среду. Никакой скрытой логики — папка, место и аргументы всегда видны.

— что запускаем
Запустить Claude профиль ccmy ▾ на этот ПК ▾ в папке 📁 ~/proj ▾ с --resume ▾
🧩
Место (SSH) и среда — независимы. Переключатель «на …» меняет только где запускается среда: «этот ПК» (локально) или «🖥 minipc» (по SSH). Поэтому можно «opencode на minipc», «Claude на этот ПК» и любые комбинации. Поле «профиль» появляется только для Claude.

2 Фраза подстраивается под среду и место

Показаны 4 состояния — видно, что лишние поля исчезают, а место/папка остаются явными.

opencode · локально
Запуститьopencode на этот ПК ▾ в📁 ~/proj ▾ --effort max ▾
нет «профиля» (он только у Claude)
shell · локально
Запуститьshell на этот ПК ▾ в📁 E:\Scripts ▾
нет профиля и аргументов — просто PowerShell в папке
Claude · по SSH (minipc) — opencode/shell тоже так умеют
ЗапуститьClaude профильccmy ▾ на 🖥 minipc ▾ в🗂 C:\Scripts ▾
папка стала «удалённой» (🗂) — пути уже на сервере
shell · по SSH = просто вход на сервер
Запуститьshell на 🖥 minipc ▾ в🗂 ~ ▾
это и есть прежнее «подключиться по SSH»
⚠️
Техническое примечание (проверим на этапе спека): «Claude/opencode по SSH» = запуск этой команды на сервере (там она должна быть установлена и авторизована). Для minipc (Windows/PowerShell) — через тот же механизм ssh -t … powershell … Set-Location, что уже сделан для удалённой папки. «shell по SSH» работает всегда.

4 Папка: проекты, избранное, недавние

Дропдаун папки — единое место выбора «где работает». Проекты = подпапки из «корня проектов» (задаётся в настройках). ★ закрепляет папку в избранное. Для SSH вместо обзора — список сохранённых удалённых путей.

5 Избранное (ярлыки запуска) и Наборы

Ярлык = вся фраза целиком, закреплённая звёздочкой ★ из конструктора. Запуск в один клик. Набор = несколько сессий разом (бывшее «Запустить все» становится просто одним из наборов).

★ Избранное: Claude · ccmy · ~/proj opencode · 🖥 minipc shell · E:\Scripts ＋ ⊞ Наборы: ▷ «работа» (4) ▷ «все профили» ＋ сохранить текущие
★
На любой запущенной панели и в конструкторе есть ★ «В избранное» — так растёт ряд ярлыков. Удаление/переименование — там же. Это закрывает «как добавлять в избранное».

6 Настройки сессий (⚙)

Всё «глобальное» собрано в одной панели за ⚙ — больше не разбросано по тулбару. Тут же «корень проектов», из которого берётся список Проектов.

📂 Корень проектов
→ подпапки = «Проекты»
📁 Папка по умолчанию
⚙ Аргументы по умолчанию Claude
▦ Колонок по умолчанию
📡 Автопроверка серверов
вкл пинговать хосты при открытии списка
✅
Это убирает прежнюю путаницу с тумблером «Спрашивать папку»: папка теперь всегда видна во фразе и подставляется по умолчанию — спрашивать отдельно не нужно. «Запустить все» переезжает в Наборы и ведёт себя предсказуемо.
