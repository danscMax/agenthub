# Castellyn — что улучшить, простыми словами (2026-06-30, v2)

Этот прогон — после визуального обхода всех экранов и проверки рабочих процессов. Большая часть «больших» багов уже разобрана (см. `report-2026-06-29.md`). Здесь — то, что осталось: **визуальные мелочи** (которые бросаются в глаза) и **функциональные пробелы** (которые мешают делать что-то из приложения без костылей).

Условные значки: 🔴 важно · 🟡 желательно · ⚪ мелочь. Работа: ⚡быстро · 🔧средне · 🏗долго.

---

## Сначала важное — безопасность и потеря данных

### 🔴 C2. Из Credential Manager нельзя удалить credentials от freellmapi 🏗⚡
Когда вводишь email/токен, они пишутся в системное хранилище Windows. Удалить или сменить — никак: кнопки «удалить credentials» нет, и сама функция в бэкенде умеет только писать, не удалять. Если сменил email — старый лежит бессрочно. *Что делать:* добавить новую команду `delete_freellmapi_auth` + кнопку в настройках провайдеров. **Что даст:** гигиена credentials, отсутствие «забытых» логинов в системе.

### 🔴 C1. «Выключил автозапуск», а в реестре всё равно осталось 🏗⚡
Жмёшь тумблер «Запускать с Windows», экран говорит «выключено» — но при следующей загрузке Windows приложение снова стартует. Обе стороны виноваты: бэкенд при «выключении» пишет `let _ = reg.delete(...).output()` без проверки результата (stderr глотается), фронт без try/catch показывает успех. **Что даст:** тумблер наконец начнёт работать как написано.

### 🔴 C3. Нет автообновления приложения 🏗🏗
Это Tauri-десктоп-приложение, но в нём **не подключён** плагин автообновления. Каждая новая версия — ручное скачивание exe. В настройках есть только текущая версия, никакой кнопки «Проверить обновления». **Что даст:** обычный user-friendly опыт desktop-приложений, всегда актуальная сборка.

## Визуальные баги — сразу бросаются в глаза

### 🟡 V2. Цвета статусов в светлой теме почти нечитаемы 🏗⚡
**Самое видимое**: главная цифра «экономия» в Аналитике блёкло-зелёная, статус задач в Расписании жёлто-красный, ошибка в конфиге профиля серая — и это в **светлой теме**. В тёмной всё ок, в светлой коэффициент контраста 1.7–2.8 (нужно ≥ 4.5). 8 файлов используют сырые Tailwind-цвета вместо уже существующих токенов. **Что даст:** в светлой теме всё читается. *Для разработчика: заменить `text-emerald-400/amber-400/red-400/500` на `status-ok/warn/bad/info` (классы уже есть в app.css).*

### 🟡 V3. Сортировка в таблицах только мышью 🏗⚡
DataTable — общий компонент, его используют 6 экранов (Форки, Sync, MCP, Профили, Провайдеры, Плагины). Сортировать можно только кликнув по заголовку. Клавиатура — никак. **Что даст:** сортировка работает с клавиатуры и со скринридера.

### 🟡 V4-V5. Доступность 🏗⚡
❶ **Уведомления (ToastHost)**: мышью навели — пауза работает, клавиатурой сфокусировали — нет (мёртвый код). ❷ **Окна подтверждения**: screen reader читает «dialog, button, button», без «Удалить профиль?» — не связаны `aria-labelledby`. **Что даст:** клавиатура и скринридер работают как ожидается.

### 🟡 V6-V10. Согласованность интерфейса 🏗🔧
❶ Смешаны две системы иконок: одни экраны рисуют квадратики `⑂ ⛁ ⧉ ☰`, другие — цветные эмодзи `🔌 🧩 💲`. ❷ В Профилях колонка «Действия» 160px, в неё запихнули 3 кнопки и дубль-кнопку «Изменить». ❸ В Форках 5 плиток KPI, часть кликабельные, часть нет — отличить нельзя. ❹ В Аналитике двойная нумерация «1. 1. name» (браузер нумерует `<ol>` + разметка вручную). ❺ Заголовки секций между табами стилизованы по-разному. **Что даст:** единая визуальная система, меньше «неровных мест».

## Функциональные дыры — что нельзя сделать из приложения

### 🟡 F9. Еженедельные архивы бэкапов невидимы 🏗⚡
Бэкап-скрипт каждое воскресенье делает `weekly.zip`, но в UI видно только счётчик `snapshots / weeklies`. Открыть, восстановить, удалить — нельзя. *Что даст:* то же, что и для обычных снапшотов — кнопки «восстановить/удалить/проверить целостность».

### 🟡 F10. Нельзя клонировать GitHub-only репозиторий из приложения 🏗🔧
В таблице «есть на GitHub, но не локально» единственная кнопка — открыть на GitHub. Хочу клонировать → через терминал. **Что даст:** одна кнопка «Клонировать сюда».

### 🟡 F11. В Настройках нет «Открыть папку бэкапов» 🏗⚡
Есть кнопки для папки скриптов, конфига, стек-файла — а папки с бэкапами нет. **Что даст:** одна кнопка → открывается проводник.

### 🟡 F12. У терминальной панели в Сессиях нет «Открыть рабочую папку» 🏗⚡
У каждой панели есть `cwd` (текущая директория), но проводник по ней не открыть — приходится копировать путь руками. **Что даст:** клик на значок папки.

### 🟡 F13. Кнопка «Открыть freellmapi» ведёт на захардкоженный порт 🏗⚡
Если поменял порт gateway в `stack.json`, кнопка всё равно ведёт на `http://localhost:13001` — 404. Бэкенд **уже умеет** динамически читать порт, просто не пробросили IPC. **Что даст:** кнопка всегда ведёт куда надо.

### 🟡 F14. Четыре опасных операции в Сессиях без подтверждения 🏗⚡
Клик по `✕` рядом с именем workspace → workspace удалён со всеми конфигами. Клик по `✕` у SSH-хоста → хост удалён. Клик по `↻` в терминале → сессия убита, scrollback потерян. Клик по `✕` у сохранённого запуска → забыт. Все четыре — **без подтверждения**. **Что даст:** стандартный диалог «Точно удалить?».

### 🟡 F15. «Отправить всем» показывает только команду 🏗⚡
Жмёшь «отправить `y` всем 8 SSH-сессиям» — диалог говорит «yes (8 panes)», но **не показывает** какие именно панели (есть ли там прод-сервер?). **Что даст:** предпросмотр списка «cc1@~/prod1, prod-server, …».

### 🟡 F16. Лимит сессий считается дважды и расходится 🏗🔧
Главное окно блокирует на 12 панелях, бэкенд — на 24 суммарно (с учётом detached-окон). После «Разнести по мониторам» можно упереться в «превышен глобальный лимит» без видимой причины. **Что даст:** счётчик показывает реальное состояние.

### 🟡 F17. Обновление 10 плагинов идёт по одному 🏗🔧
Каждый ждёт ~2 секунды; всё это время глобальный run-lock держит и ничего другого запустить нельзя. **Что даст:** либо параллельное выполнение, либо явная подсказка «skip lock».

### 🟡 F18-F20. Трей-меню бедное 🏗⚡
Только три пункта: показать окно, проверить всё, выход. Нет «Refresh forks», «Start/Stop стек», «Открыть бэкапы», «Settings». Плюс «Проверить всё» форс-джампит на вкладку Updates — теряет фокус из редактирования. Плюс «Выход» — без подтверждения, убивает PTY-сессии в фоне. **Что даст:** расширенное меню + безопасный выход.

### 🟡 F21-F22. Глобального «Отменить всё» нет; layout restore без предпросмотра 🏗🔧
❶ Нажал Esc — отменилась текущая операция, но per-repo fork и PTY крутятся дальше. ❷ При «восстановить раскладку мониторов» toast говорит «N panes на M мониторов» без списка «какой tool / какая папка». **Что даст:** ❶ «Cancel all»; ❷ предпросмотр layout перед запуском.

### 🟡 F23. На главной странице нет кнопок «починить» 🏗⚡
Home показывает «всё плохо / всё хорошо», но из него нельзя запустить «Проверить всё» или «Запустить стек» — только перейти на вкладку. **Что даст:** кнопки быстрого действия прямо на главной.

### ⚪ F24. «Открыть папку скиллов» открывает рандомный путь 🏗⚡
Берётся путь первого скилла из массива как маркер. Для opencode/coderabbit скиллов это игнорируется. **Что даст:** открывает канонический путь коллекции.
