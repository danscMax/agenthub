## Top-10 (развёрнуто, простым языком)

1. **C2 — credentials из Credential Manager нельзя удалить.** Сохранил email/token для freellmapi, потом поменял — старый лежит в Credential Manager бессрочно, кнопки «Удалить credentials» нет нигде. *Аналогия:* записал старый PIN на стикер и забыл стереть. **Фикс:** новый IPC `delete_freellmapi_auth(key)` + UI-кнопка в `ProvidersTab.svelte:540-547` рядом с `hasEmail/hasToken` бэйджами. *Что сломается:* ничего — только аддитивная команда. *Caller analysis:* `ProvidersTab.svelte:625-626` `onMyProviderDelete(p.id)` уже есть, но удаляет только JSON; добавим рядом кнопку, которая дёргает новый IPC.

2. **C1 — Autostart «выключен», но в реестре запись осталась.** Включил → выключил → пользователь думает «готово», при следующем входе в Windows приложение снова стартует. Причина: backend на off-пути делает `let _ = reg.delete(...).output()` (`lib.rs:6337-6340`) — stderr глотается, Ok возвращается всегда; плюс frontend без try/catch (`SettingsTab.svelte:209-213`). *Аналогия:* нажал «закрыть дверь», защёлка вроде щёлкнула, но дверь полуоткрыта. **Фикс:** `let status = ...; if !status.success() return Err(stderr);` в обоих ветках; на UI — try/catch + rollback (как sibling `toggleStartHidden:214-221`). *Caller:* одна кнопка `SettingsTab.svelte:368 <Toggle onCheckedChange={toggleAutostart}>`.

3. **V2 — статусные цвета нечитаемы в светлой теме.** 8 файлов используют сырые `text-emerald-400`/`text-amber-400`/`text-red-400` — на белой карточке контраст 1.7–2.8:1 (WCAG AA требует 4.5:1). Самая видимая цифра — «экономия» в `AnalyticsTab.svelte:315` — почти невидима в light mode. *Аналогия:* красный текст на розовой бумаге. **Фикс:** заменить все 8 на токен-классы (`status-ok/warn/bad/info`), которые уже определены в `app.css:273-298` с правильным `.light` override. Тривиальный глобальный swap. *Что сломается:* ничего — классы выглядят одинаково в dark, в light корректно.

4. **V3 — сортировка в таблицах только мышью.** DataTable — канонический компонент, его используют 6 табов (`ForksTab, SyncTab, McpTab, ProfilesTab, ProvidersTab, PluginsTab`). `<th>` имеет только `onclick` — keyboard и AT-пользователи не могут сортировать. *Аналогия:* пульт с кнопками, которые можно только нажать пальцем. **Фикс:** добавить `tabindex="0"` и `onkeydown` handler для Enter/Space; `<th>` уже имеет `aria-sort`. *Caller:* 6 файлов, все автоматически получат keyboard-sort.

5. **C3 — нет автообновления.** Tauri-приложение без `tauri-plugin-updater`. В Settings/About показывается только текущая версия, нет кнопки «Проверить обновления», нет prompt на release notes. *Аналогия:* телефон без автообновления приложений. **Фикс:** подключить `tauri_plugin_updater::Builder`, добавить IPC `check_for_update` / `install_update`, меню в Settings → About. L-усилие (нужно GitHub-релиз-pipeline).

6. **F9 — еженедельные архивы невидимы в UI.** Скрипт создаёт `Backups/weekly/*.zip`, бэкенд их читает (`lib.rs:835-849`), но `BackupTab` показывает только `{snapshots.length} / {weeklies.length}`. Списка weekly нет, кнопок «открыть/восстановить/удалить» нет. *Аналогия:* сейф с двумя отделениями, открывается только одно. **Фикс:** добавить в `BackupTab` второй `<section>` со списком weekly, кнопки `Restore` / `Open in Explorer` / `Verify` / `Delete` по аналогии со snapshot'ами.

7. **F10/F11/F13 — мелкие провайдер/forks разрывы.** ❶ Нет «Clone here» для GitHub-only репо (`ForksTab.svelte:303-306`). ❷ Нет «Open backup folder» в Settings About — есть только scriptsRoot/configPath/stackPath (`SettingsTab.svelte:466-476`). ❸ localhost:13001 хардкод в «Открыть freellmapi» (`ProvidersTab.svelte:531`), хотя backend уже умеет динамически читать порт через `gateway_base_url()` (`lib.rs:3347-3354`). *Фиксы:* ❶ новый IPC `clone_repo(url, target)`; ❷ добавить `backupDir` в `app_paths` + кнопку openPath; ❸ пробросить `gateway_base_url` как IPC `gateway_base_url()` — фронт уже почти готов.

8. **F14 — 4 destructive-операции в SessionsTab без confirm.** `deleteWorkspace` (`:879`), `removeFav` (`:743`), SSH `deleteServer` (`:574`), TerminalPane `relaunch` (`:367`). Все: один клик → данные потеряны / PTY убит, без `askConfirm` dialog. Для SSH-хоста — безвозвратное удаление настроек подключения. *Аналогия:* кнопка «Удалить» без предупреждения. **Фикс:** `askConfirm(danger:true, ...)` перед каждой. Шаблон уже есть в `ConfirmDialog.svelte`. L-усилие на bulk-операцию, S на каждый по отдельности.

9. **F15 — Send-to-all показывает только команду, не контекст панелей.** При нажатии «Отправить всем» появляется диалог с `cmd="yes"` и `count=8`, но не список «cc1@~/prod1, cc2@~/prod2, ssh-prod-server, …». Для 8 SSH-сессий с прод-серверами это рискованно. *Аналогия:* кнопка «отправить SMS всем контактам», без предпросмотра кому. **Фикс:** построить `details` из `panes.map(p => \`${p.tool}@${p.profile} ${p.cwd}\`)`, передать в `ConfirmDialog.details`.

10. **F16 — MAX_PANES vs SESSION_LIMIT рассинхрон.** FE блокирует добавление pane когда `panes.length >= 12` (per-window, `SessionsTab.svelte:369, 383`), BE блокирует когда global `state.0.lock().len() >= 24` (`lib.rs:6600`). Детеached-окна считают отдельно → после «Разнести по мониторам» можно не заметить, что глобально уже 22+ panes. *Аналогия:* счётчик людей в комнате не видит людей в соседней. **Фикс:** добавить IPC `global_session_count()`, в FE использовать `MAX(global + panes.length, MAX_PANES)`.
