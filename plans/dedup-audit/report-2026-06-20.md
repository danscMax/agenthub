# Castellyn — аудит дублирующихся реализаций (max-dedup, READ-ONLY)

Дата: 2026-06-20. Метод: механический проход (jscpd) + семантический sweep (3 read-only агента) →
**личная верификация на сыром коде** (вывод агентов — только лид). По каждому дублю прочитаны ОБА
места; каждое утверждение снабжено `file:line`. Код не изменялся.

> Этот аудит — НЕ копипаст-сканер. jscpd по проекту: **0.54 %** (4 клона: i18n-импорты + ScriptKit
> self-clone — ожидаемо). Вся ценность здесь — в **семантическом слое**: одна задача, решённая 2+ раза
> по-разному, с общими токенами почти без пересечения.

### 🟠 F2 — Форматтер абсолютной даты: фикс бага живёт в одной копии из четырёх — ✅
**Места реализации:**
- `src/lib/components/BackupTab.svelte:58-68` (`fmtAbs`) — **исправлено**
- `src/lib/components/ComponentCard.svelte:40-48` (`fmtTime`) — баг присутствует
- `src/lib/components/ForksTab.svelte:103-111` (`fmtTime`) — баг присутствует
- `src/lib/components/ScheduleTab.svelte:26-34` (`fmtNext`) — баг присутствует

**Общая задача:** отрендерить ISO-таймстамп как локализованную абсолютную дату, терпя null/невалидный вход.
**Что расходится:** три копии используют `try { new Date(ts).toLocaleString(loc) } catch { return ts }`.
`BackupTab` заменил это на явную проверку `if (!Number.isNaN(d.getTime()))` и оставил **комментарий**
(`BackupTab.svelte:55-57`), прямо документирующий баг: «`new Date('2026-06-08_030000')` yields an Invalid
Date WITHOUT throwing, so the old try/catch never caught it → "Invalid Date" leaked». То есть в трёх копиях
`catch` никогда не срабатывает на невалидной (но не бросающей) дате — пользователь видит «Invalid Date»
вместо прочерка.
**Проявление:** механизм бага доказан чтением (см. комментарий + код). Триггер-вход underscore-формата
кормится только в `BackupTab` (там и починили); три оставшиеся копии получают ISO-строки из статус-конверта,
поэтому **реальное** срабатывание у них маловероятно, но защита отсутствует — любой невалидный-непарсящийся
таймстамп от backend утечёт как «Invalid Date». Классический потерянный фикс.

**Blast radius:** каждая функция локальна, зовётся в разметке своего компонента
(`ComponentCard.svelte:126`, и т.д.). Консолидация в общий `fmtAbs(ts)` (дом — `src/lib/relativeTime.ts`,
рядом с `relTime`): сохранить пустой-возврат (`t('common.dash')`, не литерал `'—'`) и опциональный
snapshot-name fallback (нужен только `BackupTab`). Тестов на эти функции нет.
**Рекомендация:** канон = **логика `BackupTab.fmtAbs`** (единственная корректная). Вынести
`formatAbsTime(ts, {snapshotFallback?})` в `relativeTime.ts`, перевести 4 места.
**Стоимость:** small.
