# Полный визуальный редизайн — дизайн-спека (2026-07-02)

Решения владельца: волна 1 (группировка сайдбара) влита как есть; масштаб — **полный
визуальный редизайн, сначала спека** (это она); волна 2 — все четыре кандидата:
Home-кокпит, Updates-сводка, Providers-карточки, Sessions-тулбар.

## 0. Принципы

1. **Спокойный визуал.** Один акцент (синий бренда) + семантические статусы из
   `statusColor.ts` — и всё. Декоративные цвета из статистик убираются.
2. **Иерархия — типографикой и воздухом,** не рамками и заливками.
3. **Плотность — рабочий инструмент.** Переключатель density обязан реально менять ритм.
4. **Каждый экран отвечает на два вопроса:** «что случилось?» и «что сделать?» — без
   чтения таблиц целиком. Сводка сверху, детали по клику.

## 1. Типографический канон

| Роль | Токен | Размер | Начертание |
|---|---|---|---|
| Заголовок вкладки | `--sw-text-xl` (новый) | 1.25rem | 650 |
| Заголовок секции | `--sw-text-lg` (новый) | 0.95rem | 600 |
| Основной текст | `--sw-text-base` | 0.875rem | 400 |
| Вторичный | `--sw-text-sm` | 0.75rem | 400 |
| Подпись/капс | `--sw-text-xs` | 0.6875rem | 600, letter-spacing |

- **Запрет ad-hoc `font-size` в компонентах** — сейчас ~12 мест (2rem, 1.05rem, 0.92rem,
  0.95rem…) мимо токенов; все переводятся на канон. Nav-item → `--sw-text-base`.
- Метрики и счётчики — `font-variant-numeric: tabular-nums` (не прыгают при обновлении).

## 2. Плотность и ритм

- Все паддинги через `--sw-space-*`. Известные хардкоды (беклог 17.06): `Sidebar` nav-item
  `11px 14px`, rail `60px`, `Console`, `Select`, `DropdownMenu`, Sessions-чипы — токенизируются,
  после чего density-переключатель начинает действовать на весь chrome.
- Compact-режим: таблицы (Backup/Profiles/Updates) дают настоящие плотные строки
  (рекомендация панели), карточки уменьшают вертикальные паддинги на одну ступень.
- Секции внутри вкладок: единый `.section-title` (уже канон) + одинаковый вертикальный зазор
  `--sw-space-6` между секциями.

## 3. Цвет

- Бейджи и точки — только семантика: ok / warn / bad / info / muted.
- Forks: радуга статистики → монохромные счётчики + один сигнальный цвет для «нужны руки».
- Светлая тема наследует все изменения через токены автоматически (проверка контраста ≥3:1
  для новых поверхностей — гейт волны).

## 4. Волна 2A — Home-кокпит

Сейчас: набор равнозначных карточек; новичку не видно «с чего начать», ветерану — «что нового».

```
┌ Статус-полоса ────────────────────────────────────────────┐
│ ● Стек работает (3/3) · последнее обслуживание 2 ч назад  │
├ Быстрые действия ─────────────────────────────────────────┤
│ [▶ Новая сессия] [⟳ Проверить всё] [⛁ Бэкап] [Обновить всё]│
├ Требует внимания (агрегация attention) ───────────────────┤
│ ⑂ Форки: 1 конфликт → открыть · ⛁ Бэкап устарел → запустить│
├ Последние прогоны ────────────────────────────────────────┤
│ ✓ plugins · 2 мин · обновлено 3   ✗ forks · 14:02 · код 1 │
└───────────────────────────────────────────────────────────┘
```

- Первый запуск (нет данных): полоса заменяется ориентацией «3 шага» (путь к скриптам →
  проверить всё → открыть Сессии).
- Лента прогонов — из существующей истории `*.last.json` (без нового бэкенда).

## 5. Волна 2B — Updates-сводка

Сейчас: стена одинаковых карточек, состояние тонет.

- Хедер: **«N обновлений доступно · проверено X мин назад»** + `[Обновить всё]` (primary),
  `[Проверить]` (ghost).
- Секции по реальному состоянию: **Доступны** (строка: имя · current→new · [Обновить] · лог) →
  **Удержано** → **Ошибки** → **Актуально** (свёрнуто, только счётчик).
- Карточки → компактные строки; вся диагностика — deep-link в Run log.

## 6. Волна 2C — Providers

Сейчас: 5 крупных секций подряд, простыня.

- Верх: якорные chips-подразделы (Сервисы · Свои провайдеры · Роутер · Подключение).
- **Одна карточка на сервис**: имя + статус-точка + порт, primary-действие
  (Start/Stop/Dashboard); PID, uptime, health-детали — под `Advanced ▸` expander.
- «Свои провайдеры» — существующий `DataTable` вместо ручной секции.
- Секции сворачиваемые; состояние запоминается.

## 7. Волна 2D — Sessions-тулбар

Сейчас: ~12 контролов в один ряд без групп; глобальные и панельные действия перемешаны.

- **Глобальный ряд** (над сеткой): Поиск по всем · Команда всем + переключатель broadcast ·
  Колонки 1/2/3 · Закрыть все — с текстовыми подписями.
- **Панельные действия** — только в заголовке панели: имя/лимиты · поиск · A−/A+ · копия ·
  detach · close; редкие (очистить, скачать лог, сниппеты) — в `⋯` overflow (DropdownMenu).
- Launcher: визуальная группировка «профиль → машина → папка → аргументы» разделителями.

## 8. Порядок работ

Каждая волна: ветка → реализация → гейты (`check` 0/0, `test`, `check:i18n`, `cargo test`,
`build`) → живые скриншоты в `plans/redesign/screens/` → merge. Порядок: **1.5 (типографика/
плотность/цвет — фундамент) → 2A → 2B → 2C → 2D**. README-скриншоты обновляются один раз в
конце. Обязательный смоук: живой запуск Sessions в ?shot-харнессе (урок v0.6.0).
